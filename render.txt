========================================
RENDER DEPLOYMENT GUIDE - CchdDash
========================================

PROJECT OVERVIEW:
This is a full-stack healthcare financial dashboard application with:
- Frontend: React (Create React App with TypeScript)
- Backend: Node.js + Express + TypeScript
- Database: SQLite (embedded)
- Authentication: JWT tokens with bcrypt password hashing + MFA support
- Security: Audit logging, role-based access, password policies

TARGET DEPLOYMENT:
- Frontend: https://connect301.arkitech-test.xyz/
- Backend API: https://api.connect301.arkitech-test.xyz/

========================================
RECOMMENDED DEPLOYMENT: BLUEPRINT (EASIEST)
========================================

Your project includes a render.yaml file that automates deployment of both services.

STEPS:
------
1. Push your latest code to GitHub:
   git add .
   git commit -m "Prepare for deployment"
   git push origin master

2. Log in to Render (https://render.com)

3. Click "New +" → "Blueprint"

4. Connect your GitHub repository:
   - Repository: https://github.com/Wmartin23/CchdDash.git
   - Branch: master

5. Render will automatically detect render.yaml and show you:
   - Backend API Service (cchddash-api)
   - Frontend Static Site (cchddash-frontend)

6. Review the configuration and click "Apply"

7. Wait for both services to build and deploy (5-10 minutes)

8. Your services will initially be live at:
   - Backend API: https://cchddash-api.onrender.com
   - Frontend: https://cchddash-frontend.onrender.com

IMPORTANT NOTES:
- Both services are FREE on the free tier
- Backend will auto-generate a secure JWT_SECRET
- Database is SQLite (ephemeral on free tier - resets on restart)
- Auto-deploy when you push to GitHub
- Default user: Wmartin (ask admin for password)

========================================
CUSTOM DOMAIN SETUP (connect301.arkitech-test.xyz)
========================================

STEP 1: DNS CONFIGURATION
--------------------------
At your domain registrar (where arkitech-test.xyz is managed), add these CNAME records:

For Frontend:
  Type:  CNAME
  Name:  connect301
  Value: cchddash-frontend.onrender.com
  TTL:   3600 (or Automatic)

For Backend API:
  Type:  CNAME
  Name:  api.connect301
  Value: cchddash-api.onrender.com
  TTL:   3600 (or Automatic)

STEP 2: ADD CUSTOM DOMAINS IN RENDER
-------------------------------------
Frontend Custom Domain:
1. In Render Dashboard, go to "cchddash-frontend" service
2. Navigate to "Settings" → "Custom Domain"
3. Click "Add Custom Domain"
4. Enter: connect301.arkitech-test.xyz
5. Click "Save"
6. Wait for SSL certificate (Let's Encrypt) - usually 1-5 minutes

Backend API Custom Domain:
1. In Render Dashboard, go to "cchddash-api" service
2. Navigate to "Settings" → "Custom Domain"
3. Click "Add Custom Domain"
4. Enter: api.connect301.arkitech-test.xyz
5. Click "Save"
6. Wait for SSL certificate - usually 1-5 minutes

STEP 3: VERIFY DEPLOYMENT
--------------------------
Test Backend API:
  curl https://api.connect301.arkitech-test.xyz/api/health

  Expected response:
  {"status":"ok","message":"Server is running"}

Test Frontend:
  Open browser: https://connect301.arkitech-test.xyz/
  You should see the login page with ArkiTech branding

STEP 4: VERIFY ENVIRONMENT VARIABLES
-------------------------------------
Frontend (cchddash-frontend):
  REACT_APP_API_URL=https://api.connect301.arkitech-test.xyz

Backend (cchddash-api):
  NODE_ENV=production
  JWT_SECRET=<auto-generated-or-set-your-own>
  PORT=3005

These should already be set from render.yaml, but verify in Render Dashboard.

========================================
OPTION 2: MANUAL DEPLOYMENT (TWO SEPARATE SERVICES)
========================================

If you prefer to deploy the frontend and backend separately:

BACKEND API (Web Service):
--------------------------
1. Go to Render Dashboard
2. Click "New +" → "Web Service"
3. Connect your GitHub repository
4. Configure:
   - Name: cchddash-api
   - Environment: Node
   - Build Command: cd server && npm install && npm run build
   - Start Command: cd server && npm start
   - Instance Type: Free

5. Add Environment Variables:
   - NODE_ENV=production
   - JWT_SECRET=<generate-a-random-secure-string-32-chars>
   - PORT=3005

6. Click "Create Web Service"
7. After deployment, add custom domain: api.connect301.arkitech-test.xyz

IMPORTANT: Backend runs on PORT 3005 (not 3001)

FRONTEND (Static Site):
-----------------------
1. Go to Render Dashboard
2. Click "New +" → "Static Site"
3. Connect your GitHub repository
4. Configure:
   - Name: cchddash-frontend
   - Build Command: npm install && npm run build
   - Publish Directory: build
   - Auto-Deploy: Yes

5. Add Environment Variables:
   - REACT_APP_API_URL=https://api.connect301.arkitech-test.xyz

6. Click "Create Static Site"
7. After deployment, add custom domain: connect301.arkitech-test.xyz

========================================
TROUBLESHOOTING
========================================

DNS NOT RESOLVING:
------------------
Issue: Domain doesn't resolve to Render
Solutions:
  - DNS changes can take 5 minutes to 48 hours
  - Use: nslookup connect301.arkitech-test.xyz
  - Verify CNAME records are correct
  - Check DNS propagation: https://dnschecker.org

CORS ERRORS:
------------
Issue: Frontend can't connect to backend
Solutions:
  - Verify REACT_APP_API_URL is set correctly
  - Check allowed origins in server/src/server.ts
  - Ensure custom domain is in allowedOrigins array
  - Clear browser cache and hard reload

SSL CERTIFICATE ISSUES:
-----------------------
Issue: "Not Secure" warning in browser
Solutions:
  - Wait 5-10 minutes for Let's Encrypt provisioning
  - Verify DNS is pointing to Render correctly
  - Check Render Dashboard for SSL status
  - Try accessing via HTTPS explicitly

BUILD FAILURES:
---------------
- Check build logs in Render Dashboard
- Verify all dependencies are in package.json
- Ensure Node version compatibility (16+)
- Check for TypeScript errors

ROUTING ISSUES (404 on page refresh):
--------------------------------------
- Frontend routing is handled by the rewrite rule in render.yaml
- This should already be configured, but verify:
    routes:
      - type: rewrite
        source: /*
        destination: /index.html

DATABASE RESETS:
----------------
Issue: Users/data disappear after service restart
Cause: SQLite is ephemeral on Render free tier
Solutions:
  - Upgrade to Render paid plan for persistent disk
  - Migrate to PostgreSQL (recommended for production)
  - Accept that free tier resets on restart

SCREENSAVER ISSUES:
-------------------
- Screensaver appears after 1 minute of inactivity
- Auto-logout after 10 minutes of inactivity
- Click anywhere to dismiss screensaver
- To adjust timing, edit: src/contexts/AuthContext.tsx

GL DATA NOT LOADING:
--------------------
Issue: Financial data not appearing in dashboard components
Solutions:
  - Verify gldet.json exists in server/data/ directory
  - Check that user has 'dashboard', 'both', or 'admin' role
  - Verify JWT token is valid and not expired
  - Check browser console for authentication errors
  - Ensure backend API is running and accessible
  - Check audit logs for access denials: npm run view-logs

SPARKLINE NaN ERRORS (FIXED):
------------------------------
Issue: Console shows "polyline attribute points: Expected number, '0,NaN'"
Status: FIXED in latest version (2025-10-24)
- Sparklines now check for data variation before rendering
- No longer displays when all values are identical
- Prevents division by zero in coordinate calculations

WRONG PORT CONNECTION (FIXED):
-------------------------------
Issue: "ERR_CONNECTION_REFUSED :3001/api/gl-data"
Status: FIXED in latest version (2025-10-24)
- All components now use centralized API_ENDPOINTS config
- Backend correctly runs on PORT 3005
- GLTransactions component updated to use proper endpoint
