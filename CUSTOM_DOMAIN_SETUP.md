# Custom Domain Setup for connect301.arkitech-test.xyz

This guide explains how to deploy CchdDash to your custom domain `https://connect301.arkitech-test.xyz/`

## Architecture Overview

- **Frontend**: `https://connect301.arkitech-test.xyz/`
- **Backend API**: `https://api.connect301.arkitech-test.xyz/`

## Step 1: Deploy to Render

### 1.1 Push Code to GitHub
```bash
git add .
git commit -m "Configure for custom domain deployment"
git push origin master
```

### 1.2 Deploy via Render Blueprint
1. Go to https://render.com and sign in
2. Click "New" → "Blueprint"
3. Connect your GitHub repository (Wmartin23/CchdDash)
4. Render will detect the `render.yaml` file
5. Click "Apply" to deploy both services

### 1.3 Wait for Initial Deployment
- Backend API will initially be at: `https://cchddash-api.onrender.com`
- Frontend will initially be at: `https://cchddash-frontend.onrender.com`

## Step 2: Configure DNS Records

Add the following DNS records at your domain registrar (where `arkitech-test.xyz` is managed):

### For Frontend (connect301.arkitech-test.xyz)
```
Type:  CNAME
Name:  connect301
Value: cchddash-frontend.onrender.com
TTL:   Automatic or 3600
```

### For Backend API (api.connect301.arkitech-test.xyz)
```
Type:  CNAME
Name:  api.connect301
Value: cchddash-api.onrender.com
TTL:   Automatic or 3600
```

## Step 3: Add Custom Domains in Render

### 3.1 Frontend Custom Domain
1. In Render Dashboard, go to `cchddash-frontend` service
2. Navigate to "Settings" → "Custom Domain"
3. Click "Add Custom Domain"
4. Enter: `connect301.arkitech-test.xyz`
5. Click "Save"
6. Wait for SSL certificate to be provisioned (usually 1-5 minutes)

### 3.2 Backend API Custom Domain
1. In Render Dashboard, go to `cchddash-api` service
2. Navigate to "Settings" → "Custom Domain"
3. Click "Add Custom Domain"
4. Enter: `api.connect301.arkitech-test.xyz`
5. Click "Save"
6. Wait for SSL certificate to be provisioned

## Step 4: Verify Environment Variables

### Frontend Environment Variables
Ensure the frontend has this environment variable set:
```
REACT_APP_API_URL=https://api.connect301.arkitech-test.xyz
```

This should already be configured in `render.yaml`, but verify in Render Dashboard under the frontend service's "Environment" tab.

### Backend Environment Variables
Verify these are set in Render Dashboard for the backend service:
```
NODE_ENV=production
JWT_SECRET=<auto-generated-secure-value>
PORT=3001
```

## Troubleshooting

### DNS Not Resolving
- **Issue**: Domain doesn't resolve to Render
- **Solution**:
  - DNS changes can take 5 minutes to 48 hours to propagate
  - Use `nslookup connect301.arkitech-test.xyz` to check DNS
  - Verify CNAME records are correct

### CORS Errors
- **Issue**: Frontend can't connect to backend API
- **Solution**:
  - Check that `https://connect301.arkitech-test.xyz` is in the allowed origins list in `server/src/server.ts`
  - Verify `REACT_APP_API_URL` is set correctly in frontend environment variables
  - Check browser console for specific CORS error messages

### SSL Certificate Not Provisioning
- **Issue**: "Not Secure" warning in browser
- **Solution**:
  - Wait 5-10 minutes for Let's Encrypt to provision
  - Verify DNS is correctly pointing to Render
  - Check Render Dashboard for SSL status
  - Contact Render support if it takes longer than 1 hour

### Frontend 404 Errors on Refresh
- **Issue**: Refreshing any page except home gives 404
- **Solution**: This should already be fixed by the rewrite rule in `render.yaml`:
  ```yaml
  routes:
    - type: rewrite
      source: /*
      destination: /index.html
  ```

### Backend Not Accessible
- **Issue**: API endpoint returns 502 or timeout
- **Solution**:
  - Check Render logs for the backend service
  - Verify build completed successfully
  - Check that database initialized properly
  - Ensure PORT environment variable is set to 3001
